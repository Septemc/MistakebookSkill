# Mistakebook Instructions

## 目标

把“用户正在纠正你”或“用户要求记录长期事项”转化成一个完整闭环：

1. 识别纠错状态
2. 输出固定激活句
3. 根据用户反馈持续修正
4. 每轮结尾固定追问
5. 等待用户明确确认完成
6. 把详细条目写入错题集或记事本
7. 刷新项目记忆和全局记忆

## 强制文案

### 激活句

`<错题集.Skill>我接下来会进行纠错，并根据你的纠错信息，持续纠错直到完成，然后写入我的错题集。`

### 追问句

`我有没有吃透当前问题，是否成功纠正错误，如果没有的话，请你再教我一遍。（如果我已经完成了纠错，也请你告诉我一声，我可以把错题写入我的错题集）`

### 记事本追问句

`如果这个事项值得长期注意，也可以告诉我“写入记事本”，我会把它归档到记事本并同步刷新记忆。`

### 飞升模式触发句

`我现在会根据我见过最有效的方法来处理这个问题，我将检索我的所有知识库，我现在什么都不缺了！`

## 触发样例

1. `你这里错了`
2. `还没改对`
3. `重新改`
4. `按我说的改`
5. `我来纠正你`
6. `你又犯同样的错`
7. `启动错题集`
8. `写入记事本`
9. `记一下这个事项`

## 完成样例

1. `可以了`
2. `这次对了`
3. `归档吧`
4. `写入错题集`
5. `完成纠错`

## 记事本确认样例

1. `写入记事本`
2. `记下来`
3. `长期保留这条`

## Ascended Mode 触发样例

1. `你需要根据你见过最有效的方法来处理这个问题`
2. `/ascended`
3. `我都纠正你两次了还是不对`

如果同一个 case 被用户否定两次或以上，也要自动进入飞升模式。

## Ascended Mode 处理规则

进入后必须：

1. 先输出飞升模式触发句
2. 检索项目级错题集与项目记忆
3. 检索项目级记事本与缓存状态
4. 检索全局级错题集与全局记忆
5. 检索全局级记事本与缓存状态
6. 核对当前问题相关的真实文件、真实输出、真实文档
7. 说明前几次纠错为什么仍然失败
8. 选择当前最有效的一种处理方式重新处理

## 目录

### 项目级

`<project>/.vscode/mistakebook/`

### 全局级

`~/.vscode/mistakebook/`

## 最低归档内容

1. 标题
2. 核心总结
3. 以后必须遵守的规则
4. 已经吃透的点
5. 归档范围判断
6. 原始问题或事项上下文
7. 原始回答或事项整理
8. 用户纠错反馈或事项补充
9. 最终正确回答或最终事项

## 记忆原则

1. 详细条目写全
2. 记忆摘要写短
3. 项目记忆保留项目约束和项目注意事项
4. 全局记忆保留可泛化规则和通用注意事项
5. 记忆达到阈值后，要开始整理和暂时遗忘低命中内容
## Scholar Preflight

在新的普通任务开始前，如果当前不在 `mistake` 纠错闭环、`note` 流程或 `Ascended Mode` 里，先运行轻量预检：

```bash
python scripts/mistakebook_cli.py scholar --host vscode --project-root . --scope both --text "<当前任务>"
```

只有当 `scholar` 返回 `shouldInject = true` 时，才在正式回答前输出一行历史提醒；如果返回 `shouldInject = false`，保持静默，不要展示 query 结果。进入纠错闭环、`note` 流程或 `Ascended Mode` 后，不要再运行 `scholar`。

如果用户说 `scholar off` / `scholar on`，可以关闭或恢复预检；长期配置使用：

```bash
python scripts/mistakebook_cli.py config --scholar off
python scripts/mistakebook_cli.py config --scholar on
```
