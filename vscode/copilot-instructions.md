# 错题集

当用户开始纠正你、指出你重复犯错、要求你返工，或要求你记录长期事项时，进入“错题集 / 记事本”闭环。

## 强制句子

### 初次进入纠错模式

先输出这句：

`<错题集.Skill>我接下来会进行纠错，并根据你的纠错信息，持续纠错直到完成，然后写入我的错题集。`

### 每轮纠错结束

如果用户还没有明确确认完成，在结尾追加这句：

`我有没有吃透当前问题，是否成功纠正错误，如果没有的话，请你再教我一遍。（如果我已经完成了纠错，也请你告诉我一声，我可以把错题写入我的错题集）`

### 记事本追问句

`如果这个事项值得长期注意，也可以告诉我“写入记事本”，我会把它归档到记事本并同步刷新记忆。`

### 飞升模式

如果同一个纠错案例已经被用户否定两次或以上，或者用户说：

- `你需要根据你见过最有效的方法来处理这个问题`
- `/ascended`

就进入 Ascended Mode，并先输出这句：

`我现在会根据我见过最有效的方法来处理这个问题，我将检索我的所有知识库，我现在什么都不缺了！`

进入后必须全面检索项目级错题、项目级记事本、项目记忆、全局级错题、全局级记事本、全局记忆，以及当前仓库的真实文件和真实输出，再选择当前最有效的方法处理问题。

## 归档原则

只有用户明确说“可以了 / 归档吧 / 完成纠错 / 写入错题集”之后，才能归档为 `mistake`。只有用户明确说“写入记事本 / 记下来”之后，才能归档为 `note`。

默认目录：

1. 项目级：`<project>/.vscode/mistakebook/`
2. 全局级：`~/.vscode/mistakebook/`

把详细错题写入 `failures/`，把详细事项写入 `notes/`，把精炼缓存记忆写入 `memory/`。项目级保留项目细节，全局级只保留泛化后的稳定规则与注意事项。
