# Activation Patterns

## 应该触发错题集的场景

下面这些表达通常意味着用户正在纠正你：

1. `你这里错了`
2. `这不对`
3. `你又犯同样的错`
4. `重新改`
5. `按我说的改`
6. `我来纠正你`
7. `我教你一遍`
8. `还没改对`
9. `你没有吃透`
10. `你理解反了`
11. `这个回答有问题`
12. `这个地方你改坏了`
13. `你又把之前的要求漏掉了`
14. `启动错题集`
15. `纠正`
16. `还是有问题`
17. `请重新改`
18. `你改的有问题`
19. `/mistakebook`
20. `你需要根据你见过最有效的方法来处理这个问题`
21. `/ascended`

## 应该触发记事本的场景

下面这些表达通常意味着用户在要求记录主动事项：

1. `写入记事本`
2. `记一下这个事项`
3. `这个不是错题，但要记住`
4. `把这个注意事项记下来`
5. `这个以后都要注意`
6. `把这条规则记到记事本`
7. `这个结论值得长期保留`

## 不应该误触发的场景

下面这些情况不要自动进入闭环：

1. 用户在讨论业务问题本身，而不是纠正你或要求记录
2. 用户说“这个程序报错了”，但没有指出你的回答错误
3. 用户在做普通 review，没有表现出纠错或记录意图
4. 用户只是要求你继续下一步，而不是回头纠正或归档
5. 用户在引用别人说的话，不是在纠正你，也不是在要求记录

## 进入后的第一步

### 错题集

1. 先输出激活句
2. 再执行修正
3. 结尾追加固定追问句
4. 如果当前回复里形成了稳定注意事项，再追加记事本追问句

### 记事本

1. 先给出当前事项的整理结果
2. 结尾追加记事本追问句
3. 只有用户明确确认后才归档为 `note`

## 继续纠错的判断

如果用户后续说：

1. `还不对`
2. `你再看一遍`
3. `我不是这个意思`
4. `你还是没理解`
5. `我再教你一次`

把它们视为同一个案例的 follow-up，而不是新建案例。

## 飞升模式触发

下面任一情况都应该进入飞升模式：

1. 同一个纠错 case 的 `rejection_count >= 2`
2. 用户明确说：`你需要根据你见过最有效的方法来处理这个问题`
3. 用户输入：`/ascended`

进入后先输出固定飞升模式回复语，再进行项目级 / 全局级错题、记事本、记忆的全面检索。

## 完成归档的判断

### mistake

以下表达可视为显式完成信号：

1. `可以了`
2. `这次对了`
3. `已经纠正好了`
4. `已经吃透了`
5. `归档吧`
6. `写入错题集`
7. `完成纠错`

### note

以下表达可视为显式记事信号：

1. `写入记事本`
2. `记下来`
3. `长期保留这条`
4. `把这个事项存起来`
5. `这个以后都要注意`
