# 错题集 Skill

这是给 Codex 用的精简版错题集 Skill。目标只有一个：把“用户正在纠正你”变成一个完整闭环，而不是一句“抱歉我改一下”。

## 两句硬性文案

### 首次进入纠错模式时

先输出这句，逐字一致：

`<错题集.Skill>我接下来会进行纠错，并根据你的纠错信息，持续纠错直到完成，然后写入我的错题集。`

### 每轮纠错结束时

只要用户还没有明确确认完成，就在结尾追加这句，逐字一致：

`我有没有吃透当前问题，是否成功纠正错误，如果没有的话，请你再教我一遍。（如果我已经完成了纠错，也请你告诉我一声，我可以把错题写入我的错题集）`

### 进入飞升模式时

如果进入 Ascended Mode，必须先输出这句，逐字一致：

`我现在会根据我见过最有效的方法来处理这个问题，我将检索我的所有知识库，我现在什么都不缺了！`

## 什么时候触发

任一情况都应进入错题集模式：

1. 用户明确说你错了、没改对、又犯同样错误
2. 用户开始逐条纠正你的回答、代码或方案
3. 用户说“按我说的改”“我来纠正你”“我教你一遍”
4. 用户要求写入错题集或启动错题集

不要误触发到这些情况：

1. 用户只是讨论程序 bug，不是在纠正你
2. 用户在做普通需求补充，不是在指出你的错误

## 纠错循环

### 1. 进入

一旦确认是纠错场景：

1. 输出激活句
2. 给出修正后的回答
3. 结尾追加固定追问句

### 2. 未完成

如果用户继续说“还不对”“你再看一遍”“你还是没理解”，把这些反馈并入同一个案例，继续修正，不要提前归档。

同时维护 `rejection_count`。如果同一个案例被用户明确否定两次或以上，就自动升级到 Ascended Mode。

### 2.1 Ascended Mode

下面任一情况都必须进入 Ascended Mode：

1. 同一个案例被用户否定两次或以上
2. 用户说：`你需要根据你见过最有效的方法来处理这个问题`
3. 用户输入：`/ascended`

进入后必须：

1. 先输出飞升模式固定文案
2. 全面检索项目级错题集和项目记忆
3. 全面检索全局级错题集和全局记忆
4. 核对当前仓库里和问题相关的真实文件、真实输出、真实文档
5. 先解释前几次纠错为什么仍然失败，再给出新的最强方案

### 3. 完成

只有当用户明确说下面这类话时，才允许归档：

1. `可以了`
2. `这次对了`
3. `已经吃透了`
4. `归档吧`
5. `写入错题集`
6. `完成纠错`

## 项目级 / 全局级

默认判断：

1. `project`
   - 项目路径、项目约束、项目业务、项目流程导致的错误
2. `global`
   - 通用验证失误、通用推理偏差、通用沟通问题
3. `both`
   - 既值得项目内详细复盘，也能抽出跨项目稳定规则

判断不清时先写 `project`，有明确泛化价值再补 `global`。

## 默认目录

1. 项目级：`<project>/.codex/mistakebook/`
2. 全局级：`~/.codex/mistakebook/`

如果当前不是 Codex 项目，才退回：

1. 项目级：`<project>/.mistakebook/`
2. 全局级：`~/.mistakebook/`

## 归档要求

用户确认完成后，生成一份结构化案例，至少包含：

1. `title`
2. `summary`
3. `rules`
4. `confirmedUnderstanding`
5. `scopeDecision`
6. `scopeReasoning`
7. `correctionAttemptCount`
8. `ascendedTriggered`
9. `ascendedTriggerReason`
10. `originalPrompt`
11. `originalReply`
12. `correctionFeedback`
13. `finalReply`

### 优先用脚本

如果仓库里存在 `scripts/mistakebook_cli.py`，优先这样做：

```bash
python scripts/mistakebook_cli.py bootstrap --host codex --project-root .
python scripts/mistakebook_cli.py archive --host codex --project-root . --payload-file <temp-json>
```

### 没有脚本时

手工写入：

1. `failures/INDEX.md`
2. 单条案例 Markdown
3. `memory/PROJECT_MEMORY.md`
4. `memory/GLOBAL_MEMORY.md`

## 记忆原则

### 详细案例

保留完整上下文、纠错反馈、最终正确答案。

### 记忆文档

必须短、真、可执行。每次归档都要刷新。不要把整段聊天原文塞进去。

## 定期整理

出现任一情况时，额外做一次 full rollup：

1. 同主题累计 3 次以上
2. 新案例累计 5 条
3. 距上次集中整理超过 14 天

## 禁止事项

1. 未经用户确认，禁止归档
2. 不要只记录“错在哪里”，也要记录“已经吃透什么”
3. 不要把项目私有细节原样塞进全局记忆
